# ECP-008 — Knowledge Intelligence Runtime

## 1. Objective

> **Transform Engineering OS from document loading into semantic knowledge understanding.**

Current: Foundation Loader reads files from disk on every request.
Target: Knowledge Graph is built once, cached, and queried by the Knowledge Loader. The system understands relationships between documents, not just their content.

## 2. Success Criteria

| Criteria | Measure |
|----------|---------|
| Knowledge Graph built automatically from .ai/ metadata | No manual mapping |
| Dependency Resolution covers all assets | 100% of populated assets |
| Loading Strategy enforced | always → conditional → on-demand |
| Knowledge Cache reduces disk reads | First request: disk → cache. Subsequent: cache only |
| Knowledge Coverage | 30% → 60% |
| No filesystem access from Context Builder | Loader is single entry point |
| Knowledge Validator catches broken refs, orphans, cycles | Part of pipeline |

## 3. Deliverables

| Component | File | Responsibility |
|-----------|------|---------------|
| **Knowledge Graph** | `runtime/knowledge-graph.ts` | Build graph from metadata. Store nodes + edges. Query by domain, type, level. |
| **Knowledge Loader** | `runtime/knowledge-loader.ts` | Single entry point for all knowledge access. Replaces direct filesystem reads in Context Builder. |
| **Dependency Resolver** | `runtime/knowledge-graph.ts` (integrated) | Topological sort. Circular detection. Missing dep detection. |
| **Knowledge Cache** | `runtime/knowledge-cache.ts` | In-memory LRU cache. TTL for non-Foundation assets. Invalidation on file change (future). |
| **Knowledge Validator** | `runtime/knowledge-validator.ts` | Validates graph: broken refs, orphans, cycles, level consistency. Runs on load. |
| **Knowledge Metrics** | Integrated into ExecutionContext | Tracks: cache hits/misses, graph build time, asset count, coverage %. |

## 4. Architecture Delta

```
BEFORE (Sprint 7.3):
  Filesystem (.ai/)
      ↓ readFileSync
  Foundation Loader
      ↓ KnowledgeAsset[]
  Context Builder
      ↓ ContextPackageV1
  Prompt Assembler

AFTER (Sprint 8):
  Filesystem (.ai/)
      ↓ readFileSync (once, cached)
  Knowledge Graph       ← NEW — nodes + edges from metadata
      ↓
  Knowledge Validator   ← NEW — broken refs, orphans, cycles
      ↓
  Knowledge Cache       ← NEW — LRU, TTL, invalidation
      ↓
  Knowledge Loader      ← REFACTORED — single entry, no direct FS access
      ↓ KnowledgeAsset[]
  Context Builder        ← unchanged
      ↓ ContextPackageV1
  Prompt Assembler       ← unchanged
```

## 5. Component Contracts

### Knowledge Graph
- **Input:** `KnowledgeAsset[]` from Foundation Loader
- **Output:** `KnowledgeGraph { nodes, edges }` — queryable by id, domain, type, level
- **Dependencies:** Foundation Loader
- **Failure:** Returns empty graph on error. Context Builder falls back to flat asset list.

### Knowledge Loader
- **Input:** `{ mode, strategy, maxTokens, domains? }`
- **Output:** `KnowledgeAsset[]` — ordered, filtered, cached
- **Dependencies:** Knowledge Graph, Knowledge Cache
- **Failure:** Falls back to direct Foundation Loader call (backward compatible).

### Knowledge Cache
- **Input/Output:** `KnowledgeAsset[]`
- **Strategy:** LRU, max 50 entries. Foundation docs: 5-min TTL. Non-Foundation: 1-min TTL.
- **Dependencies:** None (pure data structure)

### Knowledge Validator
- **Input:** `KnowledgeGraph`
- **Output:** `ValidationReport { brokenRefs, orphans, cycles, warnings }`
- **Dependencies:** None (pure function on graph)
- **Failure:** Logs warnings to console + ExecutionContext. Does not block pipeline.

## 7. Risk Matrix

| Risk | Level | Probability | Impact | Mitigation |
|------|-------|------------|--------|------------|
| Knowledge Graph corrupt from bad YAML | Low | 10% | High | Metadata parser already validated. Graph skips bad nodes. |
| Circular dependency in Knowledge Graph | Medium | 20% | Medium | Topological sort detects cycles. Logs + skips. |
| Cache serves stale data after Foundation edit | Medium | 30% | Medium | TTL-based invalidation. Restart clears cache. Manual: clear on deploy. |
| Performance regression from graph building | Low | 10% | Low | Graph built once at startup + cached. <50ms on server with 22 assets. |
| Legacy Context Builder broken by new pipeline | Low | 5% | High | Full fallback to existing Foundation Loader + flat list. Backward compatible. |

## 8. Migration Strategy

```
Stage 1 (Sprint 8.1)
  Knowledge Graph + Knowledge Validator built as read-only.
  No behavior change. Runs alongside existing pipeline.
  Outputs: validation reports to console logs.

Stage 2 (Sprint 8.2)
  Knowledge Cache active. Knowledge Loader replaces direct filesystem reads.
  Context Builder calls Knowledge Loader instead of Foundation Loader.
  Behavior: identical output, faster repeat requests.

Stage 3 (Sprint 8.3)
  Knowledge Metrics integrated into ExecutionContext.
  Knowledge Coverage dashboard updated.
  Baseline Delta Report published.
```
