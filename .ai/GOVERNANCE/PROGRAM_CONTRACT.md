# Program Contract Template

> Every Phase 2 program must complete this contract before implementation begins.
> This ensures all programs are consistent with the Engineering OS kernel.

## Contract Fields

### Purpose
*Why does this program exist? One sentence.*

### Responsibilities
*What does this program do? What does it NOT do?*

### Inputs
*What does this program receive? (data, requests, events)*

### Outputs
*What does this program produce? (decisions, proposals, actions)*

### Kernel Services Used
*Which Engineering OS v1.0 services does this program depend on?*
- [ ] Organization Runtime
- [ ] Trust Runtime
- [ ] Identity Runtime
- [ ] Authorization Runtime
- [ ] Planner
- [ ] Knowledge Runtime
- [ ] Semantic Engine
- [ ] Execution Specification
- [ ] Reflection Engine
- [ ] Evidence Collector
- [ ] Knowledge Evolution

### Forbidden Actions
*What may this program NEVER do?*

### Approval Requirements
*When must this program seek Founder approval?*

### Success Metrics
*How do we measure this program's performance?*

### KPIs
*Specific Key Performance Indicators tracked over time.*
